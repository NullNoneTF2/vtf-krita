# Contributing

Thanks for your interest in improving this project.

## How to contribute

- Report bugs or suggest improvements by opening an issue.
- If you want to help with code, please open a discussion or issue first so the change can be reviewed.
- Keep changes focused and easy to review.
- If you are unsure whether a change fits the project, ask before making a large change.

## Development notes

- This project is currently an experimental plugin.
- Please be respectful of the fact that the project is just a personal thing.
- If you are changing Python or C++ code, try to keep the changes well documented.
- When possible, include a short note about why the change is needed.

## Pull requests

- Follow the existing style of the repository where practical.
- Include a short summary of the change and any relevant context.
- If the change touches build behavior or packaging, mention that clearly.

## Code of conduct

Please also read [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) before participating.
